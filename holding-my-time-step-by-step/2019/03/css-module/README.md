编写 ``css`` 是非常简单的一件事情，但是编写可维护的 ``css`` 是非常困难的，为什么编写可维护的 ``css`` 是非常困难的呢？举个例子
```
span { font-size: 10px; }
```
这个样式是全局的，没有隔离的，这行代码有可能会影响页面上其他已有的样式

业界有很对团队提出了很多方式去解决这个问题
### SMACSS
### OOCSS
面向对象 css
* 结构和设计分离
* 容器和内容分离

### BEM
Block + Element + Modifer
但是建议获取他的思想而不是全部规则
### ITCSS
### ACSS
### CCSS
### Atomic CSS
